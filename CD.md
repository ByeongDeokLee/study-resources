# CI/CD
## CI란
* **정의**: 빌드/테스트 자동화 과정. CI는 개발자를 이한 자동화 프로세스인 지속적인 통합을 의미. 커밋할 떄마다 빌드와 일련의 자동 테스트가 이루어져 동작을 확인하고 변경으로 인해 문제가 생기는 부분 없음.

## CD란
* **정의**: 배포 자동화 과정. 지속적인 서비스 제공 또는 지속적인 배포를 의미

## 종류/특징
    Jenkins(젠킨스), GitLab CI/CD(깃랩), GitHub Actions(깃허브 액션), CircleCI(서클CI), Argo CD(아르고 CD)
* **Jenkins(젠킨스)**: 가장 널리 사용되는 오픈 소스 자동화 서버로, 많은 플러그인을 통해 높은 유연성과 커스터마이징을 제공
* **GitLab CI/CD(깃랩)**: GitLab 저장소와 완벽하게 통합되어 소스 코드 관리부터 배포까지 전 과정을 관리할수 있는 올인원 도구 
* **GitHub Actions(깃허브 액션)**: GitHub에 내장되어 있어 별도 설치 없이 설정파일(.yml)을 통해 간편하게 CI/CD 파이프라인 구축 
* **CircleCI(서클CI)**: 속도가 빠르고 설정이 간편하며, 클라우드 기반으로 운영되오 별도 서버 관리가 필요 없는 도구
* **Argo CD(아르고 CD)**: 쿠버네티스환경에 특화된 GitOps 기반의 지속적 배포 도구로, 저장소이 상태를 클러스터에 자동으로 반영

## 장단점
* **장점**: 개발 속도 및 효율성 향상, 코드 품질 향상, 안정적인 배포, 빠른 피드백
* **단점**: 초기 구축 및 학습 비용, 리소스 및 비용 제한, 운영 복잡성, 벤더 종속성
