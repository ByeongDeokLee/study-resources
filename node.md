# node.js 

## 1. 정의
* **개념**: Node.js는 자바스크립트를 브라우저밖에서도 실행할 수 있게 해주는 자바스크립트 실행 환경
* **엔진**: 구글 크롬의 V8엔진을 사용하여 속도가 매우 빠름
* **역활**: 웹 페이지의 동적인 화면만 만들던 자바스크립트로 파일 시스템 제어,네트워크 서버 구축 등 가능

---

## 2. 구조 요약
### V8 
* **역활**: 웹 브라우저를 만드는데 기반을 제공하는 오픈 소스 자바스크립트 엔진
* **특징**: 속도가 매우 빠르며, node.js서버급 성능을 낼수 있게 만드는 원동력

### Libuv(비동기 처리의 심장) 
* **비동기 I/O,이벤트 루프** : Libuv를 이용하여 비동기 작업을 시스템 커널에 위임하여 동작

---

## 2. 동작원리
* **1. 싱글 스레드(메인 스레드)** : 한번에 하나의 작업만 처리(해당 처리속도는 매우빠름)
* **2. 비동기 I/O** : 앞선 작업이 끝날 때까지 멈춰 있지 않고, 비동기적으로 다른 일 계속 수행하는 방식. 대규모 네트워크 요청을 효율적으로 수행
* **3. 이벤트 루프** : 작업 순서대로 대기하지 않음. 시간이 걸리는 작업(파일 읽기, DB호출)이 들어오면 일단 넘겨주고, 다음 가벼운 요청 먼저 처리. 그 후 작업이 완료되엇다는 이벤트 발생하여 결과 반환

---

## 3. 장단점
### 장점 
* **높은 생산성** : 프론트와 백엔드 모두 JS하나 개발 가능
* **빠른 속도** : V8 엔진 덕분에  실행 속도 매우빠름
* **강력한 생태계** : NPM을 통한 방대한 라이브러리 지원
* **실시간 서비스 최적화** : NPM을 통한 방대한 라이브러리 지원

### 단점 
* **CPU 연산 취약** : 복잡한 수학 연산이나 영상편집등 CPU 소모가 큰 작업에 부적합
* **콜백지옥** : 코드를 잘못 짜면 비동기 처리가 꼬여 가독성이 떨어짐
* **싱글스레디 한계** : 싱글 스레드이여서 해당 스레드가 멈추면 서버 저네가 멈출 위험이있음
